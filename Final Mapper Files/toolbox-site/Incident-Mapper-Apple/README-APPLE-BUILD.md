# Incident Mapper Apple Build Workspace

This folder contains an Apple-first packaging project for Incident Mapper.

## What is already done
- Created Capacitor iOS wrapper project.
- Copied and normalized web app into `app/web` with `index.html` as the entry point.
- Synced web assets into native iOS project at `app/ios`.
- Added iOS-friendly export fallback: file exports now attempt native Share Sheet first on iPhone/iPad.
- Removed desktop-only launch/server artifacts from bundled web assets.

## Folder layout
- `app/web` -> web app source bundled into the mobile app.
- `app/ios` -> Xcode project generated by Capacitor.
- `app/capacitor.config.json` -> app ID and app name.

## Commands
Run from `Incident-Mapper-Apple/app`:

- `npm run cap:sync` -> copy latest web code into iOS project.
- `npm run cap:open:ios` -> open the Xcode project.

## Immediate next Apple steps
1. Open Xcode project (`npm run cap:open:ios`).
2. Set Team + Signing for your Apple Developer account.
3. Set iOS Deployment Target and app icons.
4. Build and run on a physical iPhone/iPad.
5. Archive and upload to App Store Connect.

## Notes about current behavior
- Incident data remains local to the device/browser storage unless users export files.
- Map tiles/geocoding still require internet connectivity.
- No cloud service is used in this version.
